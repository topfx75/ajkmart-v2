🐛 3. Bugs / Errors / Incomplete Functions
Frontend (Customer App)
Bug 1 — Static Map (Sabse Badi Problem)
RideTracker.tsx mein map ek static .png image hai jo /api/maps/static se fetch hoti hai. Jab rider chalega, image refresh nahi hogi automatically — sirf pin wali nai image manganee padegi. Real interactive Leaflet map nahi hai.

Bug 2 — Live location update frequency
LiveTrackMap.web.tsx Leaflet map hai lekin yeh sirf tab dikh ta hai jab component mount ho. Socket.io se rider position aati hai aur riderLivePos state update hoti hai — lekin static map image automatically reload nahi hoti is position ke saath.

Bug 3 — No-riders state pe automatic retry nahi
Jab status = no_riders ho jaye, user ko manually "Retry Search" press karna hota hai. Uber style mein automatically 2-3 minute baad retry hona chahiye.

Bug 4 — School route — empty database
School shift book karne ki poori UI hai lekin admin ne koi school routes DB mein add nahi kiye. User ko "No school routes found" milega.

Bug 5 — Bargain minimum not clearly shown
Jab user offer deta hai jo minimum se kam ho, error "Offer too low" aata hai lekin exact minimum amount clearly nahi batata front-end pe.

Bug 6 — Scheduled ride max future limit nahi
User 1 saal baad ki ride bhi schedule kar sakta hai — koi max limit nahi.

Bug 7 — Pool ride — UI exists but incomplete
isPoolRide toggle hai, API bhi hai (/pool/:groupId), lekin customer ko yeh nahi dikhta ke pool mein kaun aur hai.

Backend (API Server)
Bug 8 — Fixed dispatch radius (no expansion)
Dispatch engine sirf ek radius (default 5km) use karta hai. Agar 5km mein koi rider nahi to immediately no_riders. Uber jaisi progressive expansion nahi: 2km → 5km → 10km.

Bug 9 — Haversine 1.4x multiplier inaccurate
Jab Google/Mapbox API key nahi hoti, straight-line × 1.4 use hoti hai. Yeh flat areas mein galat estimate deta hai (urban Muzaffarabad ki seedhi sadkain).

Bug 10 — Trip OTP verification optional
Rider /verify-otp endpoint call nahi kare to bhi ride in_transit pe ja sakti hai (PATCH /:id/status). OTP otpVerified flag hai but strictly enforced nahi.

Bug 11 — Live locations no TTL
Rider jo 3 ghante se online nahi lekin live_locations table mein hai — dispatch engine use dhundhega. Last location time se filter hona chahiye (updatedAt > now - 10min).

Database
Bug 12 — School routes table empty

💰 5. Fare — Admin Rules Ke Mutabiq Theek Hai?
Haan, mostly sahi hai:

Fare = max(minFare, baseFare + distance × perKm) × surgeMultiplier
GST = fare × gstPct% (agar enabled)
Total = fare + GST

✅ Admin se base_fare, per_km, min_fare per service type configure hoti hai
✅ Surge pricing with multiplier
✅ GST optional with configurable percentage
✅ Bargain: minimum 70% of platform fare AND ≥ minFare
⚠️ Distance accuracy depends on routing provider (Google/Mapbox vs Haversine)
Issue: Agar Google Maps API key nahi add ki admin settings mein, to Haversine × 1.4 use hogi — route ke hisaab se estimate galat ho sakta hai.
