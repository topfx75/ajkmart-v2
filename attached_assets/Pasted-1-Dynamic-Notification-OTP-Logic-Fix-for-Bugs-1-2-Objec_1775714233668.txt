1. Dynamic Notification & OTP Logic (Fix for Bugs #1, #2)
Objective: OTP console se hata kar dynamic channels (SMS/WhatsApp/Email) par shift karna aur Admin control dena.
Database Task: system_settings table mein ye columns add karein:
otp_debug_mode (Boolean) - Default: false
primary_otp_channel (Enum) - ['sms', 'whatsapp', 'email', 'all']
provider_credentials (JSON) - Twilio/Msg91/SMTP keys store karne ke liye.
Logic Change: auth.ts mein direct console.log khatam karein. Ek NotificationService banayein jo check kare:
typescript
if (settings.otp_debug_mode && process.env.NODE_ENV !== 'production') {
   console.log("OTP:", otp); // Sirf dev mode mein dikhe
} else {
   // Admin preference ke mutabiq real service call karein
   await NotificationProvider.send(user.phone, otp, settings.primary_otp_channel);
}
Use code with caution.

2. Wallet Security & Race Conditions (Fix for Bugs #7, #10)
Objective: Do requests ek saath balance deduct na kar sakein.
Database Task: users table ke wallet_balance column par CHECK (balance >= 0) constraint lagayein.
Backend Task: Wallet deduction logic ko Database Transaction mein wrap karein aur FOR UPDATE (Row-level lock) use karein.
Step 1: Transaction start karo.
Step 2: User row ko lock karo (SELECT ... FOR UPDATE).
Step 3: Balance check karo.
Step 4: Update karo aur commit karo.
